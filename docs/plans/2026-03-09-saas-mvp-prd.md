# Deep Research SaaS MVP PRD（中国大陆 B2C）

## 1. 产品目标

将当前本地研究工具升级为可收费的 SaaS 产品，首版面向中国大陆个人用户，提供订阅制套餐与平台代付 AI 能力，核心目标如下：

- 建立可用的账号、订阅、配额、账单闭环。
- 控制模型调用成本并实现超额拦截。
- 提供基础运营与合规能力，支撑上线与灰度发布。

## 2. 目标用户与场景

- 目标用户：高校学生、科研人员、职场写作人群。
- 核心场景：
  - 用户注册登录后发起研究任务。
  - 系统按套餐额度扣减用量并返回报告。
  - 用户在订阅中心完成购买、续费、取消。
  - 运营可查看用户、订单、订阅并执行额度纠偏。

## 3. 商业模型与套餐

## 3.1 套餐策略（MVP）

- 免费版：
  - 月额度：50,000 tokens（综合额度）
  - 并发任务：1
  - 文件资源：5 个
  - 不支持历史回溯导出
- 基础版（月付/年付）：
  - 月额度：2,000,000 tokens
  - 并发任务：2
  - 文件资源：50 个
  - 支持会话导出
- 专业版（月付/年付）：
  - 月额度：8,000,000 tokens
  - 并发任务：4
  - 文件资源：200 个
  - 支持优先队列与高级报告样式

## 3.2 超额策略

- 执行前预扣额度，不足则阻断并提示升级。
- 执行完成后按实际消耗结算，多退少补。
- 不做按量付费，超额仅支持升级套餐后继续。

## 4. 核心流程

## 4.1 注册与订阅开通

1. 用户邮箱注册/登录（MVP 不包含第三方 SSO）。
2. 首次进入分配免费版权益。
3. 用户在订阅页发起支付并开通基础版/专业版。
4. 支付回调成功后订阅生效，额度按周期重置。

## 4.2 研究任务与配额扣减

1. 用户发起研究任务。
2. 系统校验登录态与配额余额。
3. 通过后记录预扣账本并开始研究。
4. 任务完成结算实际消耗，失败触发补偿回滚。

## 4.3 取消与降级

1. 用户取消续费后保持当前周期权益。
2. 到期自动降级到免费版。
3. 降级后按免费版规则继续使用。

## 5. 功能范围

## 5.1 In Scope（MVP）

- Supabase Auth：邮箱注册、邮箱登录、忘记密码、会话续期。
- 订阅与支付：创建订单、支付回调、订阅状态管理。
- 用量计量：任务预扣、结算、补偿、超额拦截。
- 用户中心：订阅状态、用量、账单列表。
- Admin：用户检索、订阅查询、额度纠偏、订单查询。
- 合规：条款同意记录、注销申请流程。
- 观测：支付成功率、用量消耗、毛利监控。

## 5.3 Auth MVP 边界（固定）

- 包含：
  - `/login`、`/register`、`/forgot-password`、`/reset-password` 页面。
  - 未登录访问受保护页面时跳转登录页。
- 不包含：
  - Google/GitHub 单点登录
  - 企业身份系统对接（SAML/OIDC）

## 5.2 Out of Scope（MVP 不做）

- 企业多租户与席位管理。
- 发票税务自动化与复杂财务对账。
- BYOK（用户自带 API Key）双轨计费。
- 复杂营销体系（优惠券、邀请返利）。

## 6. 验收指标

- 业务指标：
  - 支付成功率 >= 95%
  - 任务配额拦截准确率 >= 99%
  - 订阅状态同步准确率 >= 99.9%
- 工程指标：
  - 核心 API 99 线响应 < 1.5s（不含模型推理）
  - 关键链路有完整审计日志
  - 灰度发布可在 10 分钟内回滚

## 7. 里程碑

- M1（2 周）：基础设施与账号（含无 SSO 登录页闭环）
- M2（2 周）：支付订阅与配额
- M3（1 周）：运营合规与上线
