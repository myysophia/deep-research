# 论文模板导出差距分析（对标中国高校本科毕业论文）

## 1. 分析对象
- 当前导出文件：`/Users/ninesun/Desktop/论文t测试/基于 4I 理论的国产美妆品牌营销策略优化研究.docx`
- 对标样稿：`/Users/ninesun/projects/deep-research/论文撰写智能体-黑总/基于4I理论的国产美妆品牌谷雨.docx`

## 2. 总体结论
当前导出结果已经完成了从“普通报告导出”到“论文顺序导出”的第一步，但距离“可直接作为中国高校本科毕业论文提交”的标准仍有明显差距。问题不只是字体和页边距，而是同时存在以下三类偏差：

1. 内容层偏差：论文正文仍混入研究过程痕迹、占位文本和系统残留。
2. 版式层偏差：当前仍是规则拼装，不是模板文件驱动，无法与学校样稿做到高度一致。
3. 学术规范层偏差：参考文献、图表来源、章节骨架和目录联动尚未达到本科论文交付标准。

## 3. 当前导出文件的主要问题

### 3.1 正文污染严重
当前导出文件在“目录”后仍混入如下不应进入正式论文的内容：
- 封面字段重复进入正文
- 研究任务说明与研究目标原文进入正文
- 系统检索残留，如 `<QUERY ...>`
- Markdown 分隔符 `---`
- 占位提示，如“请补充关键词”“暂无参考文献，请补充后再导出”

这意味着当前“论文内容模型”和“研究过程数据”尚未彻底隔离。

### 3.2 封面与前置部分不符合高校模板形态
与对标样稿相比，当前封面仅为若干段落叠放，缺少以下模板性特征：
- 固定学校抬头或模板标题布局
- 学期信息、制表单位等固定字段
- 统一的行框、对齐线或位置锚点
- 与学校模板一致的标题尺寸、字距和空白控制

这类问题无法仅靠继续微调 `docx` 段落参数彻底解决。

### 3.3 目录与页码系统仍停留在“可用”而非“像样稿”
当前目录是通用 `TOC` 字段，可自动生成，但与高校样稿相比仍存在以下差距：
- 样式不是学校模板样式
- 目录项缩进、制表位、引导符未按模板锁定
- 正文标题与目录标题的层级映射仍受生成内容质量影响
- 当前页码体系虽区分前置部分与正文，但未完全贴合学校样稿的分页编排方式

### 3.4 正文章节骨架不稳定
当前正文依赖生成结果中的 `sections` 直接导出，没有先强制重组为本科论文常见骨架，例如：
- 绪论
- 理论基础
- 现状分析
- 问题分析
- 对策建议
- 结论与展望

因此一旦生成阶段输出了“目录”“摘要”“研究过程摘录”等内容，就会污染正文。

### 3.5 参考文献不符合 GB/T 7714
当前参考文献仍主要是：
- `[序号] 标题 + URL`

这与本科毕业论文要求的 `GB/T 7714` 顺序编码制差距很大，至少缺少：
- 作者
- 文献题名
- 文献类型标识
- 出版项
- 年份
- 访问日期（网页类）

如果继续保持现状，论文在形式审查阶段很容易被判定为不规范。

### 3.6 图表系统离“论文图表”还有一段距离
虽然当前已经能导出 Markdown 表格与 Mermaid 图，但仍缺少：
- 图题、表题与图表主体的严格位置约束
- “资料来源”或“数据来源”行
- 文内图表引用联动
- 更稳定的按章编号
- 图表跨页控制

当前更像“报告插图”，还不是“本科论文图表”。

## 4. 代码层根因

### 4.1 当前封面仍是规则拼装
见：`/Users/ninesun/projects/deep-research/src/utils/thesis-template.ts`

- `createCoverChildren` 通过多个段落手工堆叠封面字段。
- 这种方式适合快速生成，不适合复刻学校固定模板。

### 4.2 正文导出未建立“论文净化层”
见：`/Users/ninesun/projects/deep-research/src/utils/thesis-template.ts`

- `buildBodyChildren` 直接遍历 `paperDocument.bodySections`。
- 这意味着只要上游结构中夹带研究过程信息，就会原样出现在论文正文里。

### 4.3 参考文献尚未结构化
见：`/Users/ninesun/projects/deep-research/src/utils/thesis-template.ts`

- `buildReferenceChildren` 只基于 `Source[]` 进行标题与链接输出。
- 没有文献类型识别，也没有 `GB/T 7714` 规则格式化。

### 4.4 当前不是“模板文件驱动导出”
见：`/Users/ninesun/projects/deep-research/src/utils/thesis-template.ts`

- `buildTemplateThesisDocxBuffer` 本质仍是按 section 生成 Word 文档。
- 它实现的是“论文规约驱动”，不是“学校模板文件占位填充”。

## 5. 可行补齐方案

### 方案 A：继续在现有 `docx` 代码生成器上打磨
优点：
- 开发快
- 复用现有结构多

缺点：
- 很难做到与学校样稿高度一致
- 封面、页眉页脚、样式继承、目录风格都会长期存在偏差

结论：
- 可作为过渡方案
- 不适合作为“最终成稿导出”方案

### 方案 B：切换到“模板文件驱动导出”
核心思路：
- 以学校 `.docx` 模板为母版
- 保留模板中的样式、页边距、页眉页脚、目录、封面布局
- 仅向模板中填充：
  - 封面字段
  - 摘要与关键词
  - 正文章节
  - 参考文献
  - 致谢
  - 附录

优点：
- 最接近用户要的“格式高度一致”
- 可以复用学校真实模板中的样式体系
- 页眉页脚、目录、字体、段前段后等更稳定

缺点：
- 需要单独处理 `.docx` XML / 占位替换 / 结构块插入
- 开发复杂度更高

结论：
- 这是推荐方案，也是当前方向

## 6. 推荐实施顺序

### 第一阶段：建立论文净化层
目标：
- 导出前先把研究过程数据转换成“纯论文 AST”

必须完成：
- 摘要、目录、研究提示语、查询语句不得进入正文
- 占位文本不得进入导出结果
- 正文章节固定映射到本科论文骨架

### 第二阶段：接入模板文件驱动导出
目标：
- 不再自由拼装封面和样式
- 改为基于学校模板生成最终 `.docx`

必须完成：
- 模板占位符协议
- 章节插入协议
- 目录与页码保留模板行为

### 第三阶段：补齐学术规范
目标：
- 让导出结果通过形式规范审查

必须完成：
- `GB/T 7714` 参考文献格式化
- 图表标题、来源、编号规范化
- 关键词、摘要、中英文信息完整校验

## 7. 验收标准
- 导出结果中不再出现研究指令、查询串、Markdown 残留、占位提示。
- 封面、目录、页码、页眉页脚与学校模板基本一致。
- 正文结构稳定，章节不混乱、不重复。
- 参考文献满足 `GB/T 7714` 的基础规范。
- 图表具备论文格式，而非报告格式。

## 8. 当前建议
后续开发应停止继续强化“普通规则拼装导出”，转而专注两件事：

1. 先把论文正文彻底净化为结构化学术内容。
2. 再把最终导出切换到学校模板文件驱动。

只有这样，最终 `.docx` 才有机会在格式上逼近真实高校本科论文成稿。
