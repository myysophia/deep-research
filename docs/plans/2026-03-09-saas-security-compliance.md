# Deep Research SaaS 安全与合规设计（MVP）

## 1. 合规范围（中国大陆优先）

- 用户协议与隐私政策版本化管理。
- 用户同意留痕与可审计。
- 账号注销与数据删除流程可执行。
- 关键接口具备限流、签名、审计能力。

## 2. 法务文档管理

- 文档类型：
  - 服务协议（terms_of_service）
  - 隐私政策（privacy_policy）
- 版本字段：
  - `version`（例：v2026.03.01）
  - `published_at`
- 同意记录：
  - user_id
  - doc_type
  - version
  - accepted_at
  - ip_hash

## 3. 账号注销与数据删除

## 3.1 状态机

- `submitted` -> `reviewing` -> `approved` -> `executed`
- `submitted` -> `rejected`

## 3.2 执行策略

- 软删除账号 + 任务数据逻辑隔离。
- 账务与审计日志依法保留，不做物理删除。
- 删除结果写入审计日志并通知用户。

## 4. 接口安全

- 鉴权：
  - 用户侧 API 使用 Supabase 会话校验。
  - 登录体系仅启用邮箱注册登录，不启用第三方 OAuth/SSO。
  - 内部管理 API 增加管理员白名单。
- 签名：
  - 支付回调必须校验签名与时间窗。
- 限流：
  - 登录、下单、回调接口按 IP/用户双维限流。
- 输入校验：
  - 统一使用 Zod 校验请求体与 webhook 字段。

## 4.1 密码与重置策略（M1）

- 密码策略：
  - 长度 8-64 字符。
  - 至少包含字母与数字。
- 重置策略：
  - 重置链接默认有效期 30 分钟。
  - 链接过期后必须重新发起“忘记密码”流程。
  - 忘记密码接口返回统一文案，不暴露邮箱是否存在。

## 5. 审计与追踪

- 审计范围：
  - 订阅变更
  - 支付回调处理
  - 额度纠偏
  - 管理员操作
- 审计字段：
  - operator_id
  - action
  - target_type
  - target_id
  - payload（脱敏）
  - created_at

## 6. 数据安全

- 敏感数据最小化存储（仅保存业务必要字段）。
- API Key 与支付密钥只存环境变量，不入库不入前端。
- 日志脱敏（邮箱、手机号、支付单号部分掩码）。

## 7. 发布与应急

- 灰度发布：
  - 先小流量放量支付与配额链路。
- 回滚策略：
  - 新链路异常时切回旧研究入口并关闭收费入口。
- 应急预案：
  - 回调积压、误扣费、额度错账三类故障流程。

## 8. 验收标准

1. 条款同意记录可按用户与版本检索。
2. 注销申请流程全状态可追踪。
3. 支付回调验签失败可被准确拦截并告警。
4. 管理员所有纠偏操作有完整审计记录。
