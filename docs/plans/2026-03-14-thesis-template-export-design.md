# 论文模板导出设计

日期：2026-03-14

## 背景

当前项目已经具备结构化论文数据、DOCX 导出、Mermaid 图与 Markdown 表格导出能力，但导出结果更接近“研究报告式 Word”，与中国高校本科毕业论文常见成稿格式仍有明显差距。

用户目标已经明确切换为：

- 取消普通报告式 DOCX 导出
- 使用高校论文模板思路进行导出
- 结果在结构和版式上尽量接近真实本科论文

## 目标

第一版模板导出需满足：

- 输出封面、原创性声明、授权书、中英文摘要、目录、正文、参考文献、致谢、附录
- 使用模板规约驱动的固定论文结构，而不是普通报告式拼装
- 支持封面元数据录入
- 保留自动目录、表格、Mermaid 图导出能力
- Word/WPS 可直接打开

## 采用方案

第一版采用“模板规约驱动导出”：

- 参考资产：
  - `论文撰写智能体-黑总/2026届毕业论文（设计）通用模板.docx`
- 导出实现：
  - 使用 `docx` 库按模板规约重建论文结构
  - 将模板拆解为固定 section：
    - 封面
    - 原创性声明
    - 授权书
    - 中文摘要
    - 英文摘要
    - 目录
    - 正文
    - 参考文献
    - 致谢
    - 附录

该方案不再使用原先的普通报告式 DOCX 生成逻辑作为主导出路径。

## 数据模型调整

在 `PaperDocument` 中新增 `templateMeta`：

- `subtitle`
- `college`
- `major`
- `className`
- `studentName`
- `studentId`
- `advisor`
- `completionDate`
- `acknowledgements`

用途：

- 封面字段填充
- 声明/授权签名区填充
- 致谢正文填充

## 导出链路

### 入口

- `src/components/Research/FinalReport/index.tsx`
  - 增加“模板信息”入口
  - 导出时携带 `paperDocument.templateMeta`

### API

- `POST /api/export/docx`
  - 统一切换为模板论文导出器
  - 由 `buildTemplateThesisDocxBuffer(...)` 负责生成

### 导出器

- `src/utils/thesis-template.ts`
  - 固定论文分节结构
  - 中英文摘要独立 section
  - 目录使用 Word TOC 字段
  - 正文使用 heading 1/2/3
  - 表格统一转三线表
  - Mermaid 优先转图片，失败时降级为文本块

## 前端交互

新增：

- `PaperTemplateDialog`

提供字段录入：

- 副标题
- 学院
- 专业
- 班级
- 学生姓名
- 学号
- 指导教师
- 完成日期
- 致谢

## 当前实现边界

第一版已经完成模板化结构切换，但仍有以下后续优化空间：

1. 目前是“模板规约驱动”，不是对 `.docx` 二进制模板做占位填充
2. 图表编号已基础支持，但文内引用联动未完成
3. 参考文献尚未做到完整 GB/T 7714 规范化
4. 目录仍依赖 Word/WPS 更新字段以显示最终页码
5. 不同学校模板尚未参数化切换

## 下一步建议

1. 将通用模板升级为“学校模板配置”
2. 抽象 `ThesisExportProfile`
3. 增加文内引用与参考文献联动
4. 增加图表目录与附录目录
5. 基于目标样稿继续收紧版式差异
