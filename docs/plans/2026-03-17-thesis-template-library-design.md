# 论文模板库与真实模板识别设计

日期：2026-03-17

## 1. 背景

当前项目已经具备：

- 论文内容模型 `PaperDocument`
- 模板元信息 `templateMeta`
- 排版设置 `layoutConfig`
- `DOCX` 导出能力

但当前导出本质上仍是“固定规则拼装”，不支持：

- 用户上传真实高校论文样稿或空白模板
- 自动识别论文格式
- 将识别结果沉淀为可复用模板
- 在不同学校格式之间稳定切换

项目目标已经明确调整为：

- 格式正确性优先于普通导出便利性
- 支持真实模板识别，而不只是占位符模板
- 建立平台模板库与用户私有模板库
- 识别后沉淀模板资产，后续稳定复用

## 2. 用户目标

用户希望系统支持：

- 上传真实高校论文样稿
- 上传学校发放的空白模板
- 输入论文要求与补充约束
- 自动识别论文格式
- 在低置信度时做轻量确认
- 试导出预览
- 保存为“我的模板”
- 后续直接复用该模板导出论文

## 3. 设计原则

- 格式正确性优先
- 自动识别优先，低置信度确认兜底
- 规则修正优先于结果手工修补
- 模板识别一次，后续重复复用
- 平台模板与用户模板使用统一模型
- 保留当前通用导出器作为兜底能力

## 4. 核心方案

采用“模板库驱动 + 智能识别沉淀”方案：

1. 用户可以选择平台模板，也可以上传空白模板或完整样稿。
2. 系统将上传文件识别为结构化模板画像 `TemplateProfile`。
3. 系统使用论文格式规范 `FormatSpec` 作为识别先验与校验依据。
4. 导出时由 `PaperDocument + FormatSpec + TemplateProfile` 共同驱动。
5. 对高置信度模板直接导出，对低置信度模板先轻量确认。

## 5. 双层模型

### 5.1 `FormatSpec`

描述通用论文规范，用于：

- 提供识别先验
- 约束导出前校验
- 按规范族组织模板

建议包含：

- 论文类型
- 必备与可选 section
- 摘要长度约束
- 关键词数量约束
- 参考文献数量与类型约束
- 标题层级与编号约束
- 图表、页码、注释等通用规范

### 5.2 `TemplateProfile`

描述具体模板如何在 Word 中实现这些规范，用于：

- 存储模板画像
- 驱动导出
- 支持模板修正与复用

建议包含：

- 模板基础信息
- 页面规则
- section 顺序与可选性
- 样式角色映射
- 封面字段锚点
- 识别置信度
- 用户确认记录
- 版本信息

## 6. 整体架构

建议拆分为 4 层：

1. 模板资产层
   - 平台模板库
   - 用户私有模板库
   - 原始模板文件
   - `TemplateProfile`
   - 模板版本与质量评分

2. 模板识别层
   - 解析 `.docx` OOXML
   - 识别 section、样式、页码、页眉页脚、封面字段
   - 输出 `TemplateProfile`
   - 生成低置信度待确认项

3. 论文装配层
   - 将 `PaperDocument` 净化为论文 AST
   - 将内容映射到模板槽位
   - 生成模板实例化结果

4. 导出与校验层
   - 合规校验
   - 模板可用性校验
   - 试导出
   - 正式 `DOCX` 导出

## 7. 模板识别引擎

识别引擎采用“OOXML 结构解析 + 规则识别 + 置信度评分”的组合方案。

### 7.1 识别步骤

1. 解包 `.docx`
2. 抽取 `document.xml`、`styles.xml`、`numbering.xml`、`header/footer.xml`
3. 切分文档逻辑块
4. 识别样式角色
5. 识别封面字段锚点
6. 映射到统一论文结构
7. 输出置信度与待确认项

### 7.2 识别目标

- 封面
- 独创性声明
- 授权书
- 中文摘要
- 英文摘要
- 目录
- 正文
- 参考文献
- 致谢
- 附录
- 正文标题层级
- 页边距、分页、页眉页脚

## 8. 用户修正策略

用户修正的是模板规则，不是导出结果本身。

### 8.1 三层修正

1. 低成本确认
   - 只确认低置信度项

2. 规则编辑
   - 结构面板
   - 字段面板
   - 样式面板
   - 页面面板
   - 规范面板

3. 结果校正
   - 通过试导出与关键页预览验证
   - 发现问题后回到规则编辑，而不是手改成品文档

## 9. 模板库能力

模板库应具备：

- 模板分类
- 模板版本
- 模板质量评分
- 模板关键页预览
- 模板复用与复制
- 模板健康检查
- 平台沉淀精品模板

## 10. 校验机制

校验分 3 层：

1. 规范校验
   - 内容是否符合 `FormatSpec`

2. 模板校验
   - `TemplateProfile` 是否可用于导出

3. 装配校验
   - 当前论文内容能否安全落到当前模板

输出分为：

- `error`：阻止导出
- `warning`：允许导出但提示风险
- `info`：优化建议

## 11. 分阶段实施

### Phase 1：打通闭环

目标：

- 上传模板/样稿
- 识别核心结构
- 生成模板画像
- 轻量确认
- 试导出
- 保存模板
- 正式导出

### Phase 2：提升准确率与复用能力

目标：

- 平台模板库
- 模板版本管理
- 模板评分与推荐
- 页面与页眉页脚识别增强
- 模板预览与差异对比

### Phase 3：高保真复杂模板增强

目标：

- 复杂母版增强
- 复杂声明页、附录目录、图表目录
- 更严格的国标文献格式
- 更精细的分页与跨页控制

## 13. 风险与规避

### 风险 1：真实模板差异过大

规避：

- 引入 `FormatSpec`
- 引入置信度与确认机制
- 保留通用导出兜底

### 风险 2：识别正确但导出不够像

规避：

- 区分“规范正确”和“高保真还原”
- 第一阶段优先正确，后续再追高保真

### 风险 3：用户修正成本过高

规避：

- 默认自动识别
- 仅暴露低置信度项
- 使用试导出闭环验证

### 风险 4：模板规则与论文内容耦合

规避：

- 保持 `PaperDocument` 独立
- 模板只描述格式，不直接持有论文内容

## 14. 结论

推荐路线为：

- 以 `FormatSpec + TemplateProfile` 为核心模型
- 以模板库为资产中心
- 以 OOXML 结构识别为基础能力
- 以模板画像驱动导出为主路径
- 以通用导出器为兜底路径

该方案能在当前仓库基础上逐步演进实现，不需要推翻已有论文导出主链路，同时能够持续沉淀高校论文格式资产。
